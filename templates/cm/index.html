Plan de Gestión de Cambios

Justificación del Plan de Control de Cambios

Para garantizar que los servicios prestados se mantengan disponibles, funcinoales y correctos con el pasar del tiempo, es de suma importancia definir un proceso que permita gestionar la forma como se llevan a cabo cambios en los servicios de manera que se pueda garantizar el menos impacto en la prestación de servicios y la continuidad de negocio.
El plan de gestión de cambios define una serie de documentos que permiten reunir la información necesaria para poder llevar acabo los cambios en un proyecto desde que este inicia.

Definición de Cambio

Para efectos de este documento, se define como cambio a toda adición, modificación o eliminación de un item de configuración, servicio, componente de servicio y/o cualquiera de sus elementos asociados.

Proceso de Control de Cambios

El proceso de gestión de cambios cuenta con los siguiente sub-procesos:

Sub-Procesos 1 - Registro de Solicitud de Cambio 2 - Revisión de Solicitud de Cambio 3 - Evaluación de cambios y planificación 4 - Aprobación de cambios 5 - Coordinación para la implementación de cambios 6 - Evaluación de cambios y cierre 2 - Manejo de cambios de emergencia
